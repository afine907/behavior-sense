## 服务详细审查结果

### 1. behavior_core (核心库)

**评分**: F=6, P=5, A=6, M=7 → **综合 6.0/10**

**主要问题**:
- 敏感配置硬编码默认密码
- 缺少 behavior_core 专用单元测试
- 模型输入验证不足
- Pydantic v2 配置语法需更新

**改进建议**:
1. 使用 `SecretStr` 类型处理敏感配置
2. 添加模型字段验证器
3. 补充单元测试覆盖

---

### 2. behavior_mock (模拟器服务)

**评分**: F=7, P=6, A=6, M=7 → **综合 6.5/10**

**主要问题**:
- 全局可变状态导致并发安全问题
- `send_batch()` 非真正批量发送
- 场景状态转换逻辑缺陷
- 缺少单元测试

**改进建议**:
1. 使用依赖注入替代全局状态
2. 利用Pulsar异步批量特性
3. 添加场景状态机验证

---

### 3. behavior_stream (流处理服务) ⚠️

**评分**: F=5, P=4, A=4, M=7 → **综合 5.0/10**

**主要问题**:
- 内存泄漏风险 - 全局状态字典无限增长
- 缺少背压处理机制
- 使用 `print` 替代 `logging`
- 状态存储重复与不一致
- 异常处理过于宽泛

**改进建议**:
1. 实现背压处理配置
2. 替换内存状态为 Faust Table
3. 引入结构化日志
4. 细化异常处理

---

### 4. behavior_rules (规则引擎服务) ⚠️

**评分**: F=6, P=5, A=5, M=6 → **综合 5.5/10**

**主要问题**:
- **`eval()` 注入风险** - 高危安全漏洞
- 全局单例竞态条件
- 热更新原子性问题
- CORS配置过于宽松
- HTTP客户端资源管理问题

**改进建议**:
1. 替换 `eval()` 为 AST 解析
2. 使用线程安全的单例模式
3. 实现"先加载后替换"热更新策略
4. 复用 HTTP 客户端连接

---

### 5. behavior_insight (洞察服务)

**评分**: F=6, P=5, A=5, M=7 → **综合 5.75/10**

**主要问题**:
- Upsert 操作存在竞态条件
- N+1 查询问题
- SCAN 全量遍历性能问题
- CORS 安全配置问题
- 测试 Mock 不完整

**改进建议**:
1. 使用 PostgreSQL `INSERT ... ON CONFLICT UPDATE`
2. 使用 Redis pipeline 批量操作
3. 缓存统计结果
4. 完善 Mock 测试实现

---

### 6. behavior_audit (审核服务)

**评分**: F=6, P=4, A=6, M=7 → **综合 5.75/10**

**主要问题**:
- `get_stats()` 执行8次独立数据库查询
- `batch_assign()` 串行处理
- 全局变量导致测试困难
- 缺少数据库复合索引
- 缺少输入验证

**改进建议**:
1. 合并为1-2次聚合查询
2. 使用 `asyncio.gather()` 并发处理
3. 改用依赖注入模式
4. 添加复合索引

## 模块化设计评估

### 当前架构分析

```
┌─────────────────────────────────────────────────────────────┐
│                     服务依赖关系图                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  behavior_core (共享内核)                                    │
│  ├── 模型定义: UserBehavior, UserProfile, UserStat...       │
│  ├── 配置管理: settings.py                                  │
│  └── 工具函数: logging, datetime                            │
│         ↓                                                    │
│  ┌──────┼──────┐                                            │
│  ↓      ↓      ↓                                            │
│ mock  stream  rules                                          │
│  │      │      │                                            │
│  └──────┼──────┘                                            │
│         ↓                                                    │
│  ┌──────┼──────┐                                            │
│  ↓             ↓                                            │
│ insight     audit                                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 模块化问题清单

| 问题 | 说明 | 建议 |
|------|------|------|
| 职责边界模糊 | behavior_core 既包含模型定义又包含配置和工具 | 拆分为 models/config/utils |
| 服务间同步调用 | rules → insight/audit 直接API调用 | 改为消息队列异步通信 |
| 共享模型耦合 | 所有服务共享同一模型定义 | 考虑服务独立模型 |
| 缺少API网关 | 无统一入口，服务调用无治理 | 引入API网关 |
